# Health Tracking Project Summary

## Project Context
This project serves as an automated health tracking and analysis pipeline designed specifically for logging and finding correlations between meals, supplements, symptoms (like cramping and bloating), and daily cycle phases (for Endometriosis/Adenomyosis and SIBO management). 

Daily data is captured via handwritten whiteboard photos, which are processed by AI subagents to extract structured data into a local SQLite database for deeper correlation analysis.

## Project Goals
1. **Digitize Handwritten Logs:** Seamlessly convert daily whiteboard tracking images into structured data.
2. **Centralize Data:** Maintain a robust, easily queryable `health_tracker.db` SQLite database containing tables for `daily_logs`, `meals`, `supplements`, and `symptoms`.
3. **Data Visualization & Tracking:** Automatically generate unified summary tables and a comprehensive Master Excel Tracker for easy viewing.
4. **Correlation Analysis:** Periodically run analysis scripts across the database to identify triggers for symptoms like bloating and cramping (e.g., food triggers, sleep disruptions, incomplete bowel movements, cycle phases).

## Workspace Architecture & File Linking
The workspace has been organized into dedicated folders to streamline the ingestion of future images:

### `/images`
- **Purpose:** The entry point. Drop all new daily whiteboard tracking photos into this folder. 

### `/scripts`
- **Purpose:** The engine. Contains all the Python and SQL code that powers the project.
- **`schema.sql` / `init_db.py`**: Used to define and reset the underlying database tables.
- **`load_data.py`**: Reads extracted JSON data and inserts it securely into the SQLite database.
- **`generate_tracker.py`**: Connects to the database and exports the `Master_Health_Tracker.xlsx` and `symptoms_chart.png`.
- **`unified_script.py`**: Combines data from all tables to print a unified markdown summary table.
- **`analyze.py` & `food_analyze.py`**: Scripts designed to calculate correlations between the different health metrics.

### `/data`
- **Purpose:** The storage. 
- **`health_tracker.db`**: The core SQLite database where all historical health data lives.
- **`data_batch_*.json`**: Raw JSON extraction files generated by the AI before being loaded into the database.

### `/outputs`
- **Purpose:** The deliverables.
- **`Master_Health_Tracker.xlsx`**: The final, updated Excel workbook containing all data split by tabs.
- **`symptoms_chart.png`**: Visual graph of symptom volume and severity over time.
